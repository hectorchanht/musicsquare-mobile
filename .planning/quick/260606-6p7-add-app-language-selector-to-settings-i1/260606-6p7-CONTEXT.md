# Quick Task 260606-6p7: App-language selector + whole-app i18n - Context

**Gathered:** 2026-06-05
**Status:** Ready for planning

<domain>
## Task Boundary

Add an **App language** selector to the settings page and **internationalize all static UI-chrome strings** across the whole app. Switching the language updates the interface **reactively** and the choice **persists** like other settings.

Scope = ~74 hardcoded strings across **12 `.svelte` files** (`src/routes/**` + `src/lib/components/**`).
</domain>

<decisions>
## Implementation Decisions (LOCKED — user answered "No preference" → all recommended defaults)

### Mechanism — custom lightweight runes `t()` (NO new dependency)
- New `src/lib/i18n/` module:
  - `index.ts` — `AppLang` type, the `t(key, params?)` helper (dot-keyed lookup + `{name}`-style param interpolation), a dict registry, and a missing-key fallback (fall back to `en`, then to the raw key — never render blank).
  - `en.ts`, `zh-Hant.ts`, `zh-Hans.ts` — per-locale dictionaries, identical key sets.
- **Reactivity:** `t()` reads `settings.appLang` (a Svelte 5 `$state` rune) on each call, so any template/`$derived` calling `t('…')` re-renders when the language changes (same reactivity model as `names.dn()` + `rev`). No page reload.
- SSR-safe (no `window`/`navigator` at module load; detection guarded by `browser`). Matches the existing store/service code style. Do NOT add svelte-i18n / Paraglide / any library.

### Locales — English + Traditional + Simplified Chinese
- Ship `en`, `zh-Hant` (繁體中文), `zh-Hans` (简体中文). English is the source/reference locale (current UI text).

### Default + detection
- Default `en`. On **first visit only** (no saved `appLang`), auto-detect from `navigator.language` (`zh-TW`/`zh-Hant`→`zh-Hant`; `zh-CN`/`zh-Hans`/`zh`→`zh-Hans`; else `en`). After that, always respect the saved choice.
- Persist `appLang` in the existing settings store (`openmusic:settings:v1`) — add field + load/save, mirroring `lyricsLang` etc.

### Scope
- Translate ALL visible chrome: body text, button labels, **toasts**, **menu item labels** (TrackMenu), empty states, input **placeholders**, and **aria-labels** (accessibility).
- **SKIP `src/routes/spike/+page.svelte`** — internal dev/egress harness, not user-facing.
- **Keep `nameLang` / `lyricsLang` independent** — those translate *dynamic content* (song/artist names, lyrics) via the network translate API. The new App language controls *UI chrome only*. Do NOT couple them (switching UI language must not change content-translation settings).
</decisions>

Use sensible dot-namespaced keys (e.g. `nav.library`, `menu.playNext`, `toast.shareCopied`, `settings.appLanguage`).
