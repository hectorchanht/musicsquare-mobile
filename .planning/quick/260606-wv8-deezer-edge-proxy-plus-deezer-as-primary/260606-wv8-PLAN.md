<objective>
Add a Deezer edge proxy and make Deezer the PRIMARY cover source for home discovery tiles.

New cover chain (final, exact — implement verbatim):
- Track tile cover: Last.fm `item.image` (cheap pre-check, already real + filtered) -> CAA-by-mbid (`caaReleaseGroupCover`, cheap pre-check) -> BACKFILL: Deezer cover -> CN-source cover (searchAll -> dedupeBest[0].cover) -> gradient.
- Artist tile cover: Last.fm `a.image` (cheap pre-check, today always null) -> CAA-by-mbid -> BACKFILL: Deezer artist picture -> gradient.

The synchronous `tileCover()` render order in `+page.svelte` is UNCHANGED at the cheap pre-check layer (Last.fm image -> CAA -> cached -> gradient). What changes is the BACKFILL that fills the gap: it now resolves Deezer-first, then CN (track) and Deezer (artist). iTunes is REMOVED from the chain entirely. Last.fm stays the cheap `item.image` pre-check in tileCover — it is the third tier of the user's "Deezer -> CN -> Last.fm" intent, satisfied at the render layer, not a backfill network call.

Purpose: Deezer has far stronger Western + decent CN album-cover coverage and (unlike iTunes / Last.fm) real artist-picture coverage, so the home shelves fill with real art instead of gradients. The v7k iTunes fallback is replaced.

Output: a new no-secret Deezer edge proxy (`/api/deezer/search`) mirroring the Last.fm proxy posture, a thin `deezer.ts` client, a reworked Deezer->CN cover-backfill, and the iTunes module deleted.

COVERS ONLY: do NOT add Deezer charts/discovery or artist/album-info consumption — those are SEPARATE later tasks (3b, 3c). The proxy MAY be shaped to extend later (a single reshape function), but only the cover/search path ships now.
</objective>
