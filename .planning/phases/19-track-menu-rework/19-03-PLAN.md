<objective>
Fix MENU-03 / D-12: after a long-press opens the track menu, the trigger element under the finger must leave NO stuck `:active` / `:hover` / focus highlight, and must not fire a second play/navigate action on finger-up. This lands entirely at the long-press TRIGGER SITES (home tiles, library/search/artist/album rows, NowPlaying queue/related rows) and one global CSS pass — NOT inside TrackMenu.

Purpose: The codebase has NO global `-webkit-tap-highlight-color: transparent` and NO `@media (hover: hover)` guard today; trigger elements use sticky `.tile:active{transform:scale(.96)}` and `.row:hover{background}` that visibly stick under a held finger on touch (iOS sticky `:hover`, persisted `:active`, grey tap-highlight flash). The trailing-click half is ALREADY handled by `longpress.ts`'s `suppressNextClick`; the missing half is the stuck visual.

Output: A global tap-highlight reset in `app.css`, each trigger site's hover rule wrapped in `@media (hover: hover)`, and a `blur()` call on each `onlongpress` handler.
</objective>
