<objective>
Introduce the `API_BASE` fetch seam (D-03) and add Capacitor-origin CORS to every `/api/*` route via a single server hook (D-02). These are pure web-repo changes that leave the deployed web build byte-identical (no env var set → same-origin relative fetch, web CORS unchanged) while making the codebase ready for the native build to call the deployed Pages `/api/*` cross-origin.

Purpose: Without this seam, the native APK's WebView (origin `https://localhost`) would request `https://localhost/api/...` (no server in the APK → 404) and the deployed routes would reject its cross-origin requests. This plan is the web-safe foundation every later native plan depends on.

Output: `api-base.ts` + tests, migrated call sites, `hooks.server.ts` CORS seam, extended allowlist + proxy tests.
</objective>
