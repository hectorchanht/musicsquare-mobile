## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → `/api/qq/*`, `/api/kuwo/*` | Untrusted query crosses into the proxy (route validation already enforced by 01-01) |
| client adapter ← upstream JSON | Untrusted QQ/Kuwo response bodies normalized into Track — contract-drift surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05a | Tampering | QQ upstream body (shape drift / HTML error page) | mitigate | Preserve the dual-format guard verbatim; throw typed contract-drift on neither-shape (Task 1 Test 3) instead of trusting the body |
| T-01-05b | Tampering | Kuwo upstream body (`code!==200`) | mitigate | Throw on non-200 code / missing data (Task 2 Tests 2 & 4) rather than normalizing a poisoned body |
| T-01-03 | Tampering (XSS) | QQ/Kuwo titles/artists/lyrics | mitigate | Adapters return plain strings into Track; never `{@html}`. Svelte auto-escapes when these render (Phase 4 / spike). No interpolation of source HTML here |
| T-01-06 | Information Disclosure | QQ `pay` / paywall signal mishandled | accept | `pay` is captured as a plain string for Phase-4 UX; it is non-secret upstream metadata, no disclosure risk |
</threat_model>
