# Phase 999.1 Plan 03: Platform-Switched Blob Store (Web IDB / Native Filesystem) Summary

Swapped the offline-blob backend behind a `Capacitor.isNativePlatform()` branch (D-10): the web
path keeps the existing IndexedDB store byte-for-byte, while native writes downloaded audio Blobs
to an app-private filesystem dir via `capacitor-blob-writer` + `@capacitor/filesystem` — same
`put/get/del` signatures, same never-throws posture, so all 4 consumers compile and pass untouched.
The public `Music/` MediaStore upgrade (D-11, resolved `public-music-mediastore`) is recorded in
code with a `// TODO(999.1-06)` marker and delivered by the device-gated follow-up plan.

## What Was Built

**Task 1 — Platform-switch blob-store backend + native-branch tests (D-10) [TDD]**

- Installed and pinned `@capacitor/filesystem@8.1.2` and `capacitor-blob-writer@1.1.20` (both
  slopcheck `[OK]`/Approved in RESEARCH — ionic-team + diachedelic, mature; no checkpoint needed).
- `src/lib/services/blob-store.ts`: added a single `if (Capacitor.isNativePlatform()) return
  native…(uid…)` line at the top of `put`, `get`, and `del`. When NOT native the existing
  IndexedDB code path runs verbatim (`openDb`, the `DB_NAME/STORE/VERSION` consts, and `txStore`
  are unchanged). When native:
  - `nativePut` streams the Blob to `Directory.Data` under `downloads/${sanitized-uid}` via
    `write_blob({ path, directory, blob, recursive: true })` — no base64 round-trip (avoids the
    +33% bloat + memory spike `@capacitor/filesystem.writeFile` would incur on large lossless files).
  - `nativeGet` reads back via `Filesystem.readFile` (base64), `atob`→`Uint8Array`→`Blob`; also
    handles the web-shim shape where `readFile` hands back a `Blob` directly; returns null on miss.
  - `nativeDel` deletes via `Filesystem.deleteFile`, swallowing not-found.
  - Every native path is wrapped so it resolves `false`/`null`/void and NEVER rejects — exact
    parity with the IDB branch (T-999.1-09: a failed download degrades to CDN playback, no crash).
- `export const blobStore = { put, get, del }` and all three signatures are byte-identical, so
  `player.svelte.ts` (x4 call sites), `library.svelte.ts`, `TrackMenu.svelte`, and the
  `player.svelte.test.ts` mock are all untouched and green.
- `// TODO(999.1-06)` marker on the native write path records the resolved public-Music handoff
  (D-11, `public-music-mediastore`, 2026-06-12) — the decision is staged, not re-asked.
- `src/lib/services/blob-store.test.ts` (new, mirrors `deezer.test.ts` header + mock posture):
  13 cases covering the namespace export shape, the web branch (isNativePlatform false → IDB path,
  native backend NOT touched), native put/get/del happy paths (write_blob/readFile/deleteFile
  invoked with the expected path + `Directory.Data`), empty-uid short-circuits, and every native
  error path resolving the sentinel without rejecting.
