<objective>
Rework `src/lib/components/TrackMenu.svelte` to satisfy MENU-01 (always-visible buttons + gated resolve-then-act), MENU-02 (two-row marquee header + top-right like/close + matching skeleton), and QUEUE-04 (Remix row). Purpose: Today the menu hides EVERY button behind a 9-row skeleton until a stub resolves, and the header is a single ellipsised `{title} · {artist}` line with no explicit close or like-in-header. This rework inverts the stub behavior (buttons first, data fills in), restructures the header to the Spotify/YT-Music sheet convention, and adds Remix — all reusing existing primitives (no new queue mechanism, no new overlay registration).

Output: A reworked TrackMenu with a 2-row marquee header, a top-right Like+Close cluster, always-rendered action rows, three gated resolve-then-act rows (Download / Detail / Remix) with inline spinners, and the old 9-row skeleton replaced by a header-only skeleton.
