## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params (?tab, ?playlist) → library state | attacker-influenceable navigation params drive tab/playlist selection |
| swipe commit → library/queue mutation | gesture mutates the user's own library/queue |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-10 | Tampering (param injection) | library ?tab / ?playlist deep-link | mitigate | `?tab` validated against VALID_TABS, `?playlist` validated against existing playlist ids; unknown values fall back to default tab without crash. Acceptance: unknown tab/playlist param falls back safely (no thrown error). |
| T-23-11 | Tampering | swipe commit (queue/like) | accept | Commit only mutates the user's own queue/library via the same paths TrackMenu uses; reversible (re-swipe to unlike); no privilege boundary. |
| T-23-12 | Information Disclosure | reveal/toast text | accept | Reveal icons + reused toast keys are static UI; no PII or secrets exposed. |
</threat_model>
