# Quick Task 260607-ii6 — Album / library / playback / NP polish

Decisions in CONTEXT.md (locked Playback approach = hidden Advanced accordion).

## Task 1 — Album-page fixes (single file)
**File:** `src/routes/(app)/album/[name]/+page.svelte`
- **Per-button disable**: replace `albumBusy: boolean` with `busyAction: AlbumAction | null`
  where `AlbumAction = 'play' | 'download' | 'like' | 'addToPlaylist' | 'share'`. Each
  handler sets/clears its own. Each button's `disabled` = `busyAction === '<self>'`.
- **Long-press track rows**: add `use:longpress` + `onlongpress={() => openMenu(stub)}` to
  the `.row` button. `openMenu(stub)` runs `resolveStub(stub.artist, stub.title)` → opens
  `TrackMenu` with the resolved Track. Loading flag prevents a second open during resolve.
- **Album-like heart fill state**: `$derived allLiked` = "resolved tracks exist AND every
  resolved track is in `library.liked`". The like button renders
  `<Heart fill={allLiked ? 'currentColor' : 'none'} />`. Same handler keeps the
  liked-missing-vs-unlike-all branch from hvu; toasts now use `t('toast.liked')` and the
  new `t('toast.unliked')`.
- **Download CORS fallback**: in the per-track save loop, when `fetch(url)` throws, fall
  back to `window.open(full.audioUrl, '_blank')` (mirrors TrackMenu doDownload's catch
  branch). Counts "saved or opened" toward the final toast.
