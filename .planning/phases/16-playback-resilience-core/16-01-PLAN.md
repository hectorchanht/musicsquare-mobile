<objective>
Collapse the repeat control from 3 states (off / one / all) to exactly 2 states (off / repeat-one), remove every reference to `'all'` from the player store and NowPlaying UI, and migrate any persisted `repeatMode: 'all'` to `'off'` at restore time. Auto-generated up-next (already shipped via `ensureAhead`/`regenerate`) is the semantic successor of repeat-all, so continuation no longer needs a repeat-all wrap.

Purpose: PLAY-10. This plan also narrows the `repeatMode` TYPE first so the resilience-core plan (16-02) builds against `'off' | 'one'` rather than the obsolete tri-state — interface-first ordering.
Output: A 2-state repeat in `player.svelte.ts` + `NowPlaying.svelte`, a one-line restore migration, and tests proving the toggle cycle + migration.
</objective>
