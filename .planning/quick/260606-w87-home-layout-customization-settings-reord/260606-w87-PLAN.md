<objective>
Add a Home-layout customization settings page (`/settings/home`) that lets the user
reorder + show/hide the home discovery sections, pick which discovery tags/countries
appear, set items-per-shelf, choose the default landing tab, pick tile density, and
toggle the home chrome (search pill / Randomize button). All config rides the existing
`settings` runes store with non-destructive localStorage migration, and **defaults
preserve today's behavior exactly** — a returning user sees no change.

Purpose: Give phone users control over their discovery surface (YouTube-Music/Spotify-style
layout config) without a new dependency, while guaranteeing first-paint and home render are
never broken by a corrupt/old persisted value.

Output:
- New pure, unit-tested helper module `src/lib/services/home-layout.ts`
- New `dragReorder` action (touch-capable vertical reorder; mirrors `dragScroll`)
- New `/settings/home` settings sub-page (registered in the settings groups + i18n × 3)
- Home + (app) shell consume the config (order/hide, subset, shelf size, density, chrome,
  landing-tab redirect)

OUT OF SCOPE (do NOT touch): light/dark THEME system; wiring the existing-but-unapplied
`accent` to a CSS variable beyond what `applyTheme()` already does. Theme is a separate
later quick task.
