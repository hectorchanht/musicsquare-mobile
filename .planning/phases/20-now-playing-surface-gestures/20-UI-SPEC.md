# Phase 20 — UI Design Contract

> Visual and interaction contract for the now-playing gesture phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase type: INTERACTION / MOTION, not a new screen.** This phase layers native-feeling gestures onto EXISTING surfaces (the `.cover`, the `.np-top` wrapper, the `.panel` scroller, and the `Nowbar` mini-player). There is no new visual layout to design from scratch. The contract below therefore locks the **interaction and motion** specifications that `20-CONTEXT.md` deliberately left to discretion, and reuses the established Phase 15 / Phase 17 motion vocabulary verbatim. Visual tokens (spacing/typography/color) are inherited unchanged from the existing app and re-stated here only as the canvas these gestures move within — do NOT introduce new tokens.

This phase ships **no new visible copy**. Every gesture is silent and the only feedback is motion + the existing loader. The NP-04 loader reuses the existing `aria-label` already on the nowbar's progress affordance. No empty state, no error state, no CTA, no toast is introduced.

| Element | Copy |
|---------|------|

| Empty state body | n/a |
| Error state | n/a — a play failure surfaces through the EXISTING `player.error` path (`.np-error` / nowbar `.err`), unchanged by this phase |

Accessibility: the `.cover` keeps `role="button"` + `tabindex="0"` + its `aria-label`; new gesture branches must not strip these so keyboard prev/next (the existing `ArrowLeft`/`ArrowRight` window handler) and screen-reader semantics survive.

## Motion & Gesture Contract

> **This is the load-bearing section for Phase 20.** Pin every value. Values marked **[REUSE]** are extracted from existing code and MUST match it byte-for-byte; values marked **[NEW]** are this phase's only genuinely new motion and are specified here as the contract.

### Shared motion vocabulary (extracted — REUSE, do not reinvent)

| Token | Value | Source |
|-------|-------|--------|
| Snap/settle easing | `cubic-bezier(.22,1,.36,1)` | NowPlaying.svelte:653, :749; swipeRemove.ts:123; dragClose.ts:91 — the app's universal "settle" curve |
| Snap/settle duration | `0.28s` (280ms) | same — used for sheet snap, swipe-remove spring-back, drag-close snap-back |
| Cover reflow duration | `0.32s` (320ms) | NowPlaying.svelte:839 (`.cover` width/height/margin/radius reflow on sheet state change) |
| NP enter/exit | `fly { y: 600, duration: 320, easing: cubicOut }` | NowPlaying.svelte:651 |
| Engage slop | `8px` (`DRAG_SLOP` / `DRAG_START` / `SLOP`) | NowPlaying.svelte:376; dragClose.ts:38; swipeRemove.ts:41 |
| Flick commit velocity | `0.5px/ms` (`FLICK_V`) | dragClose.ts:40; swipeRemove.ts:42; via `createVelocityTracker()` (3-sample) |
| Vertical collapse threshold | `120px` | NowPlaying.svelte:404; dragClose.ts default |
| Reduced motion | honor `@media (prefers-reduced-motion: reduce)` AND `:root[data-reduce-motion]` (app.css:92) | app.css — kills transitions app-wide; gesture springs must degrade to instant snap, the loader keyframe slows to 2.2s |

### 1. Cover carousel peek — prev/next (NP-01 / D-01) **[NEW motion]**

The committed cover follows the finger 1:1 on the X axis; the adjacent `queue[i-1]` (drag right) or `queue[i+1]` (drag left) album art peeks in from the opposite edge. Render up to 3 covers (prev / current / next) in a horizontal track during the drag; snap to the committed track on release.

| Property | Value | Notes |
|----------|-------|-------|
| Current-cover travel | 1:1 with finger `dx` (no damping when a neighbor exists) | matches swipeRemove's live-follow idiom |
| Neighbor layout | **full-bleed, no gutter** | YT Music / Spotify feel (D-01, specifics). Neighbors sit edge-to-edge at exactly `±coverWidth` so the next cover's leading edge meets the current cover's trailing edge with zero gap. Cover width = `min(72vw, 320px)` (NowPlaying.svelte:839). |
| Neighbor parallax | **none — 1:1 lockstep** | The 3-cover track translates as one rigid strip (`translateX(dx)`); neighbors move at the same rate as the current cover. (No sub-rate parallax — keep it a true carousel, not a stacked-card peek. Simplest, most-native, and avoids a second easing variable.) |
| Neighbor opacity during drag | `1` (fully opaque, no fade) | a peeking album cover is the affordance; fading it would read as a disabled state |
| Neighbor scale during drag | `1` (no scale) | flat carousel, not a cover-flow |
| Off-strip overflow | clip to the cover container (`overflow: hidden` on the carousel track wrapper); the third cover beyond the drag direction stays off-screen | |
| Commit settle | `transform 0.32s cubic-bezier(.22,1,.36,1)` — animate the strip so the committed neighbor lands centered, THEN swap `player.next()`/`prev()` updates the strip to the new current | use **0.32s** (the cover-reflow duration) NOT 0.28s, so the carousel settle matches the cover's own motion personality; both share the universal `.22,1,.36,1` curve |
| Spring-back (below threshold, neighbor exists) | `transform 0.28s cubic-bezier(.22,1,.36,1)` to `translateX(0)` | the app's standard snap-back (swipeRemove.ts:123 parity) |

### 2. Rubber-band resist at true queue boundaries (D-02) **[NEW motion]**

Applies ONLY where there is genuinely no neighbor to reveal — in practice only **"prev" on the very first track** (next almost always has an auto-generated up-next neighbor). Note `player.prev()` already restarts the song when `currentTime > 3`, so the resist case is narrow: prev, on track index 0, with `currentTime ≤ 3`.

| Property | Value | Notes |
|----------|-------|-------|
| Resistance curve | logarithmic / asymptotic damping: `offset = sign(dx) · maxPull · (1 − e^(−|dx| / maxPull))` | classic iOS rubber-band; finger travel asymptotes toward `maxPull`, never reaching it |
| `maxPull` (max drag offset) | `0.18 × coverWidth` (≈ 48–58px on a 320px cover) | enough to feel the wall, small enough that it never reads as a real swipe |
| Spring-back duration | `0.32s` (320ms) | match the cover-reflow personality (heavier surface = the slightly-longer settle), curve `cubic-bezier(.22,1,.36,1)` |
| Outcome | **no track change** — always springs back to `translateX(0)` | flick velocity is IGNORED at a true boundary (a hard flick into a wall must not commit) |
| Distinguishing feel | the resisted edge feels "stuck"; a real edge (with a neighbor) feels "free" | this asymmetry IS the affordance that tells the user there's nothing there |

### 3. Commit threshold (NP-01 / NP-05 / D-08) **[NEW basis, REUSE mechanics]**

Reuse `swipeRemove.ts` / `velocity.ts` mechanics; change ONLY the distance basis from flat 96px to proportional.

| Surface | Engage | Distance commit | Flick commit | Source of fraction |
|---------|--------|-----------------|--------------|--------------------|
| Cover | 8px slop, axis-locked (`|dx| > |dy|`) | **`dx ≥ 0.28 × coverWidth`** (28% of `min(72vw,320px)` ≈ 90px on a 320px cover) | `|velocity| ≥ 0.5px/ms` with `|dx| > 8px` always commits | D-08 says ~25–30%; **pinned to 28%** — large enough to avoid accidental skips on a wide cover, small enough to feel light |
| Nowbar | 8px slop, axis-locked | **`dx ≥ 0.28 × nowbarContentWidth`** (28% of the slide-able mini-player content width) | `|velocity| ≥ 0.5px/ms` with `|dx| > 8px` always commits | same 28% fraction so both surfaces feel identical (D-08 rationale) |

> **Same fraction on both surfaces** keeps the feel consistent (D-08). Flick (0.5px/ms) always wins over distance on BOTH surfaces — except at a true boundary (§2), where flick is ignored.

#### Commit feedback (D-08 "armed" hint)

**Positional only — no color, no haptic-here, no ring.** On the cover, "armed" is communicated by the peeking neighbor having crossed the cover's horizontal midpoint: once `|dx| ≥ 0.28 × coverWidth` the user can see more of the neighbor than of the current cover, which reads natively as "let go and it switches." No additional visual hint is added (consistent with the app never tinting during a drag). On the nowbar, the content has slid ≥28% of its width and is visibly leaving — same positional cue.

### 4. Tap cover → collapse to closed (NP-03 / D-04) **[REUSE]**

In the **half** state, a sub-slop tap on the cover collapses the sheet to `closed` (the immersive cover view, no subnav panel). Reuse the EXISTING closed-snap path — do not author a new animation.

| Property | Value | Notes |
|----------|-------|-------|
| Trigger | sub-slop tap (movement `< 8px` on BOTH axes) on `.cover`, only when `sheetState === 'half'` | must reach the cover's `onclick`/tap path — never `setPointerCapture` on `pointerdown` (ROADMAP Pitfall 7) |
| Animation | the existing grip-down → `closed` snap | sheet transform `transform 0.28s cubic-bezier(.22,1,.36,1)` (NowPlaying.svelte:749) + the `.cover` reflow `0.32s cubic-bezier(.22,1,.36,1)` (NowPlaying.svelte:839) running concurrently as they already do on state change |
| No-op states | `closed` (already closed) and `full` (D-04 scopes this to half only) | tap on the cover in full/closed keeps existing behavior |

### 5. Nowbar slide-and-snap — prev/next (NP-05 / D-06, D-07) **[NEW motion, lighter than cover]**

The mini-player CONTENT (thumb art + text, `.np-open` row) slides with the finger and springs back, or slides off and is replaced, on commit. **No multi-cover carousel** — the art is too small to read a peek (D-06).

| Property | Value | Notes |
|----------|-------|-------|
| Content travel | 1:1 with finger `dx` while dragging | matches swipeRemove live-follow |
| Optional fade | content opacity may fade toward `0` across `FADE_DISTANCE = 120px` of travel | OPTIONAL, mirrors swipeRemove.ts:43/105; keep subtle — D-06 is "track & snap-back," fade is secondary |
| Commit (slide off + swap) | slide the content fully off in the drag direction over `0.28s cubic-bezier(.22,1,.36,1)`, fire `player.next()`/`prev()`, then the new track's content appears (no fly-in needed — the store swap repaints it) | the lighter surface uses the 0.28s snap, not the 0.32s cover settle |
| Spring-back (below threshold) | `transform 0.28s cubic-bezier(.22,1,.36,1)` to `translateX(0)`, opacity → `1` | swipeRemove.ts:123 parity |
| Boundary | same rubber-band rule as §2 (only "prev" at index 0) but with the lighter `0.28s` spring-back; no peek to show anyway | |
| Tap-to-expand preserved (D-07) | sub-slop tap still fires `.np-open` `onclick` → `player.expand()` | same no-capture-on-pointerdown invariant; horizontal drag past slop changes track, vertical/sub-slop does not |
| Loader rail unaffected | the `.np-prog` rail at the top of the bar is OUTSIDE the sliding `.np-open` content — it must NOT slide with the swipe | keep the loader visually pinned while content slides |

### 6. Top running-line loader (NP-04) **[REUSE markup]**

Reuse the nowbar's exact loader: `.np-prog.indet > i.sliver` markup + the `np-indet` keyframe, driven by `player.loading`.

| Property | Value | Notes |
|----------|-------|-------|
| Markup | `<div class="np-prog indet"><i class="sliver"></i></div>` — copy from Nowbar.svelte:40–48 | the SAME class names so it inherits the keyframe; do not author a new loader |
| Mount position | **at the very top edge of the `.np` view, full-bleed across the width, BELOW the iOS safe-area inset** (under `env(safe-area-inset-top)`) and ABOVE the cover/`.bar` header | NP-04: "at the very top of the now-playing view." Anchor it so it sits flush with the notch-safe top edge, not under the notch. |
| Height | `3px` | `.np-prog { height: 3px }` (Nowbar.svelte:182) — match exactly |
| Track (rail) color | `rgba(255,255,255,0.12)` | `.np-prog` background — match exactly |
| Sliver color | `var(--color-primary)` (`#7c5cff`) | `.np-prog > i` background — match exactly |
| Sliver width / motion | `35%` width, `animation: np-indet 1.1s ease-in-out infinite` | match exactly; reduced-motion slows to `2.2s` (the existing override) |
| Visibility condition | shown whenever `player.loading === true` | `loading` set true at player.svelte.ts:~1131, false at :~1296 |
| Shows in which sheet states | **both `half` and `full` AND `closed`** — i.e. always at the NP view's top whenever NP is mounted and loading | Claude's discretion (D / NP-04). Rationale: the loader communicates track-load progress, which is state-independent; pinning it to the view's top edge means it is visible the instant any track starts resolving regardless of sheet height. It is positionally distinct from (and does not duplicate) the nowbar's own `.np-prog` when the embedded Nowbar appears in `full` — if visual duplication is observed in `full`, suppress the top loader in `full` only (since the embedded Nowbar already carries one). Default: show in all states; suppress-in-full is the only permitted fallback. |
| z-index | above the cover and `.bar` header, below any modal/overlay | so a loading track's progress is never occluded by the cover image |

### 7. Scroll containment (NP-02) **[behavioral contract]**

`overscroll-behavior-y: contain` on the scrollable `.panel` (NowPlaying.svelte:901, `.panel { flex:1; overflow-y:auto }`) so scroll/bounce stops at the panel's edges and never chains to the page behind the half-open sheet.

| Property | Value | Notes |
|----------|-------|-------|
| Rule | add `overscroll-behavior-y: contain;` to `.panel` | front-layer scroll never reaches the page behind it |
| Confirm against | both `half` and `full` layouts | the panel is the scroller in both; verify the half layout's panel actually overflows so the rule has effect |
| `touch-action` | keep the panel's vertical pan with the browser (`pan-y` if a horizontal action is ever attached to a child) | do NOT set `touch-action: none` on the panel — it must scroll |

## Cross-Browser / Platform Constraints (mobile-first)

| Constraint | Impact | Contract |
|------------|--------|----------|
| iOS Safari `overscroll-behavior` support | Supported iOS 16+; pre-16 the rubber-band page bounce may still leak | NP-02 is best-effort on iOS < 16; the `.panel` rule is still correct and is the documented mechanism. Do not add JS scroll-locking as a workaround in this phase. |
| `setPointerCapture` + trailing click (Pitfall 7) | Capturing on `pointerdown` retargets the trailing click and kills tap-to-play / tap-to-collapse / tap-to-expand | **Never** `setPointerCapture` on `pointerdown`; capture only in `pointermove` after slop + axis dominance; sub-slop movement must reach `onclick` (the load-bearing invariant for cover D-05 and nowbar D-07; matches swipeRemove.ts / dragClose.ts) |
| `touch-action` axis yield | The cover's existing `.np-top { touch-action: pan-x }` (NowPlaying.svelte:838) lets the browser own horizontal pan | Reconcile: the NEW horizontal cover swipe needs the horizontal axis. Set the gesture container so the action owns X (the existing comment says "none here, but…" — the new swipe is exactly the case it anticipated). The vertical collapse continues to claim Y after slop. Do not let the two race — arbitrate by axis dominance (`|dx|>|dy|` → carousel; `|dy|>|dx|` → collapse). |
| iOS momentum scroll vs horizontal swipe on `.panel` | a horizontal swipe inside the scrollable panel could fight scroll | the carousel swipe lives on `.cover` / `.np-top`, NOT on `.panel`; the panel stays `pan-y`. No collision by construction. |
| Android Chrome | full support for `overscroll-behavior`, pointer events, the velocity idiom | no special handling |
| Reduced motion | OS pref + the app's `:root[data-reduce-motion]` setting | carousel/rubber-band/nowbar springs collapse to instant snap; loader keyframe slows to 2.2s (existing override). Track change still happens — only the animation is removed. |
