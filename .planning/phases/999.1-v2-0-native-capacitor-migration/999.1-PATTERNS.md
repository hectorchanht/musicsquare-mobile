## Pattern Assignments

### `src/lib/services/api-base.ts` (NEW — service, request-response) — D-03

**Analog:** `src/lib/services/deezer.ts` (thin never-throws client: module-level path consts + a single fetch funnel through one helper). Use it for the **module-doc-comment + path-const + single-fetch-funnel** convention, NOT its caching (api-base is a pure URL prefixer).

**Doc-comment + consts convention** (`deezer.ts:1-25`, condensed): every `src/lib/services/*.ts` opens with a block comment stating posture (never-throws, no secret crosses boundary, own-origin proxy), then module-level `const X_PATH = '/api/...'`. Mirror that header style.

**Recommended new module** (from RESEARCH Pattern 3, lines 226-232):
```ts
// src/lib/services/api-base.ts — NEW
// API_BASE seam (D-03): web build leaves VITE_API_BASE unset → '' → same-origin relative
// fetch (unchanged). Native build bakes VITE_API_BASE=https://openmusic.pages.dev at build
// time via Vite import.meta.env → all /api/... calls become absolute cross-origin.
const BASE = import.meta.env.VITE_API_BASE ?? '';
export function apiUrl(path: string): string { return BASE + path; }
export function apiFetch(path: string, init?: RequestInit): Promise<Response> {
  return fetch(apiUrl(path), init);
}
```

**Migration surface — call sites that must route through `apiUrl()`/`apiFetch()`** (from RESEARCH lines 219-223; re-grep at plan time to confirm line numbers):
- Source adapters: `src/lib/sources/{netease,qq,kuwo,joox,fivesing,jamendo}.ts`. **Pattern they use today** (`netease.ts:46`): `const url = '/api/netease/search?...'; const res = await fetch(url, { signal });`.
- **Pitfall 3 — `<audio>.src` rewrite (correctness):** `netease.ts:87` and `:90` set `track.audioUrl = '/api/netease/url?...'` / `track.lrcUrl = '/api/netease/lrc?...'` as **relative** strings consumed directly by `<audio>.src`. These MUST go through `apiUrl()` or the APK requests `https://localhost/api/...` (no server in the APK) → silent Netease. All other sources set an absolute CDN `audioUrl`, so only Netease is affected (verify with grep at plan time — RESEARCH A6).
- Services with path consts: `lastfm.ts`, `translate.ts`, `deezer.ts` (consts at `:28-33`), `similar.ts`, `cover-backfill.ts`, `discovery.ts`, `itunes-cover.ts`.

### `src/lib/proxy/http.ts` (MODIFY — utility, request-response) — D-02

**Analog:** itself. Append the Capacitor origins to the existing `ALLOWED_ORIGIN_PATTERNS` array.

**Existing pattern** (`src/lib/proxy/http.ts:9-19`):
```ts
const ALLOWED_ORIGIN_PATTERNS: RegExp[] = [
  /^https:\/\/openmusic\.pages\.dev$/,            // deployed app (D-06)
  /^https:\/\/[a-z0-9-]+\.openmusic\.pages\.dev$/, // CF preview deploys
  /^http:\/\/localhost(:\d+)?$/,                  // local dev
  /^http:\/\/127\.0\.0\.1(:\d+)?$/
];
function isAllowedOrigin(origin: string | null): origin is string {
  return !!origin && ALLOWED_ORIGIN_PATTERNS.some((re) => re.test(origin));
}
```
**Add** (RESEARCH Pattern 4, lines 240-244 — note existing `/^http:\/\/localhost(:\d+)?$/` already matches `http://localhost`, so only the `https`/`capacitor` schemes are new):
```ts
/^https:\/\/localhost$/,      // Capacitor Android default (server.androidScheme: 'https')
/^capacitor:\/\/localhost$/   // iOS / legacy capacitor scheme (harmless to allow now)
```
**Critical posture (Anti-Pattern, RESEARCH 262 + `http.ts:5-7`):** NEVER `Access-Control-Allow-Origin: *` — the proxy carries the JOOX token. The `corsHeaders(origin)` echo-only-if-allowlisted pattern (`http.ts:26-36`) and `Vary: Origin` must be preserved (Cloudflare edge cache correctness).

### `src/hooks.server.ts` (NEW — optional single CORS seam) — D-02 (Claude's discretion: hooks vs per-route)

**Analog:** `src/routes/api/[source]/[...path]/+server.ts` (the OPTIONS + `corsHeaders` pattern). There is **no `hooks.server.ts` in the repo** (verified) — clean add if the planner chooses the hook approach.

**Existing per-route OPTIONS/CORS pattern to lift into a hook** (`src/routes/api/[source]/[...path]/+server.ts:57-60`):
```ts
// CORS preflight — scoped to the own origin via corsHeaders (never `*`).
export const OPTIONS: RequestHandler = ({ request }) => {
  return new Response(null, { status: 204, headers: corsHeaders(request.headers.get('origin')) });
};
```
**Why a hook may be needed (verify-each-route gate, RESEARCH 236):** the catch-all (`/api/[source]`) applies `corsHeaders` on GET + OPTIONS, but the **dedicated routes** (`/api/deezer/*`, `/api/lastfm/*`, `/api/translate`, `/api/similar`) each wire their own — the `deezer/search/+server.ts` header confirms it "mirrors the /api/lastfm/discovery posture VERBATIM (own-origin CORS, OPTIONS 204 preflight)". **At plan time, grep every `/api/*/+server.ts` for `corsHeaders` + an `OPTIONS` export**; any route missing it will reject the Capacitor origin. A `hooks.server.ts` `handle` that adds allowlisted CORS headers and answers OPTIONS for all `/api/*` in one place removes that per-route audit risk.
**Cloudflare gotcha (RESEARCH 245):** the hook runs on the Pages Function (workerd) — OPTIONS must return a `Response` and not fall through to downstream logic; keep `Vary: Origin`.

### `src/lib/services/blob-store.ts` (MODIFY — service, file-I/O) — D-10

**Analog:** itself. The module is **already the seam**: a `put`/`get`/`del` interface plus a `blobStore` namespace export. Native filesystem slots behind the same signatures.

**Existing never-throws + lazy-cached-open contract to preserve** (`src/lib/services/blob-store.ts:18-47, 57-113`):
```ts
import { browser } from '$app/environment';
let openPromise: Promise<IDBDatabase | null> | null = null;
function openDb(): Promise<IDBDatabase | null> {
  if (!browser) return Promise.resolve(null);
  if (typeof indexedDB === 'undefined') return Promise.resolve(null);
  if (openPromise) return openPromise;          // lazy + cached; failure caches null
  /* ... resolve(null) on every error path, never reject ... */
}
export async function put(uid: string, blob: Blob): Promise<boolean> { /* false on any failure */ }
export async function get(uid: string): Promise<Blob | null> { /* null on miss/error */ }
export async function del(uid: string): Promise<void> { /* silent on success/miss/failure */ }
export const blobStore = { put, get, del };     // consumed as `import { blobStore } from '$lib/services/blob-store'`
```
**Platform switch (D-10):** branch on `Capacitor.isNativePlatform()` (from `@capacitor/core`). Web branch = the existing IDB code UNCHANGED. Native branch = `capacitor-blob-writer` → `@capacitor/filesystem` write, with the SAME `put/get/del` signatures and the SAME never-throws posture (resolve `false`/`null`/void, never reject). **Keep the `blobStore` namespace export identical** — 4 consumers depend on it: `src/lib/stores/player.svelte.ts` (`:363,:433,:1301,:1395` all call `blobStore.get(uid).catch(() => null)`), `src/lib/stores/library.svelte.ts`, `src/lib/components/TrackMenu.svelte`, `src/lib/stores/player.svelte.test.ts`. Do not change the signatures or the callers break.
**Storage-dir decision (Pitfall 2 / D-11):** public `Music/` requires a non-npm MediaStore plugin (Android 10+); the zero-dep fallback is `Directory.Data`/`External` (app-private). RESEARCH flags this for `checkpoint:human-verify` before committing to the MediaStore plugin — the planner must gate it.

### `src/lib/services/media-session.ts` (MODIFY — keep pure) + `player.svelte.ts` bridge — D-05

**Analog:** the file itself + the `player.svelte.ts` `ms` accessor — this is an **exact existing seam**.

**Keep the PURE helpers untouched** (`src/lib/services/media-session.ts:35,54,66`): `buildArtwork(cover)`, `safePositionState(duration, position)`, `playbackStateFor(hasTrack, playing)` are node-Vitest-testable and platform-agnostic. They stay as-is and `media-session.test.ts` keeps passing.

**The single guarded accessor that must become a platform branch** (`src/lib/stores/player.svelte.ts:540-558`):
```ts
private get ms(): MediaSession | null {
  return typeof navigator !== 'undefined' && 'mediaSession' in navigator
    ? navigator.mediaSession
    : null;
}
private syncPosition(el: HTMLAudioElement) {
  const ms = this.ms; if (!ms) return;
  const st = safePositionState(el.duration, el.currentTime);
  if (st) ms.setPositionState(st);
}
private syncPlaybackState() {
  const ms = this.ms;
  if (ms) ms.playbackState = playbackStateFor(!!this.current, this.playing);
}
```
**Bridge approach (RESEARCH Pattern 5, lines 249-258):** the Android System WebView has **no `navigator.mediaSession`**, so on native this accessor returns null today and the OS media UI is a silent no-op. Introduce a `Capacitor.isNativePlatform()` branch: on native, route `setPositionState`/`playbackState`/metadata/action-handlers to the chosen plugin (jofr's API is Web-MediaSession-shaped by design, minimizing the diff). **Critical (RESEARCH 258):** you MUST explicitly `setPlaybackState('playing')` AND register ALL action handlers (play/pause/next/prev) wired to `player.next()/pause()/...`, or the notification never appears. The plugin choice (`@jofr/capacitor-media-session` vs `@capawesome-team/capacitor-android-foreground-service`) is a `checkpoint:human-verify` device spike (Pitfall 1).
**D-06 (do NOT regress):** the `attach()` `visibilitychange` flush near `player.svelte.ts:688+` is resume-safe (persists position, does not reload) — keep it. Never `invalidateAll()`/navigate on `App.resume`.

## Shared Patterns

### Never-throws service posture
**Source:** `src/lib/services/blob-store.ts:7-16` (doc comment) + `src/lib/services/deezer.ts:11-25`.
**Apply to:** `api-base.ts` (the migrated source adapters already swallow/throw deliberately — preserve each one's existing contract; `netease.ts:111` swallows lyric errors, `netease.ts:54` throws on contract drift), and the native `blob-store.ts` branch (resolve sentinel, never reject).
Every `lib/services/*.ts` opens with a posture block comment and either never throws or throws a single typed contract-drift error. Match that.

### Own-origin CORS (never `*`)
**Source:** `src/lib/proxy/http.ts:26-36` (`corsHeaders` echo-only-if-allowlisted + `Vary: Origin`).
**Apply to:** the `http.ts` allowlist edit AND any new `hooks.server.ts`. Forbidden: `Access-Control-Allow-Origin: *` (the proxy carries the JOOX token — open relay = T-01-02). Always echo the matched origin, set `Vary: Origin`.

### SSR / platform feature-detection guard
**Source:** `src/lib/services/blob-store.ts:27` (`if (!browser) return ...`) and `src/lib/stores/player.svelte.ts:540-544` (`ms` accessor SSR + feature guard).
**Apply to:** native branches in `blob-store.ts` and the `media-session` bridge — gate on `Capacitor.isNativePlatform()` and fall back gracefully (web behavior unchanged when not native, no-op under SSR).

### Module-level path constants
**Source:** `src/lib/services/deezer.ts:28-33` (`const PROXY_PATH = '/api/deezer/search'` etc.).
**Apply to:** `api-base.ts` consumers — relative `/api/...` strings stay as path consts; only the final `fetch` is funnelled through `apiUrl()`/`apiFetch()`.

### Vitest unit-test convention (mock-global, never-throws asserts)
**Source:** `src/lib/services/deezer.test.ts:1-23` and `src/routes/api/proxy.test.ts:66-107` (`fakeEvent` factory + `vi.stubGlobal('fetch', ...)`).
**Apply to:** `api-base.test.ts`, `blob-store.test.ts`, and the `proxy.test.ts` extension. No live network; `afterEach(() => { vi.restoreAllMocks(); vi.unstubAllGlobals(); })`.
