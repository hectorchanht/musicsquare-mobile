<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → `/api/*` (`+server.ts` on CF Worker) | Untrusted client input (`params.source`, `params.path`, query) crosses into the proxy |
| `+server.ts` → upstream proxies | Server injects the JOOX secret (01-03) and forwards; untrusted upstream JSON returns |
| build output → client bundle | The JOOX token must never cross into anything shipped to the browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing/Tampering | `params.source` in `+server.ts` | mitigate | Validate `params.source` against `PROXIES` registry; return 404 for unknown source (Security V5) — implemented Task 3 |
| T-01-02 | Elevation/DoS | `/api/*` open-proxy abuse (free CORS/music relay) | mitigate | `corsHeaders()` scoped to OWN origin; never `Access-Control-Allow-Origin: *` (Anti-Patterns line 341, Security V4) — Task 3 |
