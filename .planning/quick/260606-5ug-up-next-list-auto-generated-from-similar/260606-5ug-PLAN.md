<objective>
Make Up-Next smarter in three ways, all locked in CONTEXT.md:

1. **Similar-vibe generation** — when the user clicks a NEW song to play, regenerate the auto portion of Up-Next from songs similar in vibe/genre to that track (Last.fm `artist.getSimilar` -> search each similar artist's top track -> dedupe -> exclude current+manual). Manual entries are preserved. `next()`/`prev()`/auto-advance never regenerate.
2. **Reorder** — drag a far-right `GripVertical` handle on Up-Next rows (custom pointer/touch drag, not native DnD) to reorder the queue; the dragged track is pinned as manual.
3. **Current-track length** — elapsed / total mm:ss around the now-playing progress bar (current track only).

Purpose: a more musically coherent, user-curatable queue that survives regeneration, plus visible track length — without leaking the Last.fm key and without adding a `duration` field to `Track`.

Output: a new Last.fm proxy endpoint + service module (both tested), origin-tracking + regen + reorder in the player store, and the grip/length UI in `NowPlaying.svelte`. Works end-to-end with NO key configured (same-artist fallback).
</objective>
