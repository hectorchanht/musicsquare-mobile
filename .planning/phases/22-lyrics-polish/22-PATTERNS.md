**Core pattern to WIDEN — `splitParenLines`** (`lrc.ts:62-84`, the function being modified for D-07/D-08):
```typescript
export function splitParenLines(lines: LyricLine[]): LyricLine[] {
	const out: LyricLine[] = [];
	const parenRe = /[(（]([^()（）]+)[)）]/g;          // ← WIDEN to full bracket set (D-07)
	for (const line of lines) {
		const matches = [...line.text.matchAll(parenRe)];
		if (!matches.length) { out.push(line); continue; }
		const stripped = line.text.replace(parenRe, '').replace(/\s+/g, ' ').trim();
		// If the WHOLE line is one paren clause + nothing else, don't split — render as-is.
		if (!stripped) { out.push(line); continue; }     // ← D-09 whole-line passthrough: KEEP this branch
		out.push({ time: line.time, text: stripped });
		for (const m of matches) {
			const inner = (m[1] ?? '').trim();
			if (inner) out.push({ time: line.time, text: inner, fromParen: true });
		}
	}
	return out;
}
```
**What changes (D-07/D-08, never-drop guarantee):**
- Widen `parenRe` to the 9-pair set `（）()【】[]［］「」『』〈〉《》`. RESEARCH Pitfall 4 warns: a naive class `[（(...]( ... )[）)...]` mismatches open-with-wrong-close. Use a pair-aware alternation or a class only if a mixed-bracket fixture proves it. RESEARCH-suggested regex: `/[（(【［「『〈《]([^）)】］」』〉》]+)[）)】］」』〉》]/g` — but plan must add the mixed-bracket fixture (`愛（love）【chorus】`) to lock pairing.
- Add the D-08 script-mismatch gate per match: only push `{ fromParen: true }` when `dominantScript(inner) !== dominantScript(stripped)`. Same-script clause → leave inline (do NOT split it out; it stays part of `stripped`). This is the structural never-drop-original guarantee.
- KEEP the existing `if (!stripped)` whole-line branch unchanged (D-09).

**NEW pattern — `reorderPairs`** (insert between `parseLRC` and `splitParenLines`; consumed BEFORE split per RESEARCH Pattern 1, verified at `lrc.ts:77-81`):
```typescript
// Skeleton from RESEARCH Pattern 3 (D-04/D-05):
function reorderPairs(lines: LyricLine[]): LyricLine[] {
	// 1. Compute song-dominant script across ALL line bodies (the "original language").
	// 2. Walk same-`time` groups; within a group, if a line matches songDominant and a
	//    sibling does not, move the matching (original) line to the front.
	// 3. Pure-CN (songDominant 'han', no mismatched siblings) → group unchanged.
	// 4. Stable: preserve relative order of non-reordered lines.
}
```
- Reorder operates on WHOLE lines (must precede split — RESEARCH Pattern 1 / Pitfall 2). After reorder, first-of-group = original, so `NowPlaying.svelte` anchor logic (`activeIndexAndTime`, below) needs ZERO change (D-04).

**No error handling / no try-catch** — these are pure deterministic transforms matching the existing `parseLRC`/`splitParenLines`, which have none. Defensive guards are inline (`if (!txt) return []`, `(m[1] ?? '')`).

### `src/lib/stores/player.svelte.ts` — `seekFraction` (store, request-response) — READ ONLY, NO EDIT

**Consumed as-is** (`player.svelte.ts:1810-1840`). It is the only seek API; it clamps to [0,1], parks a `pendingSeekFrac` when metadata isn't loaded, and — load-bearing for D-03 — auto-plays when `audio.paused`:
```typescript
seekFraction(frac: number) {
	this.abortFade();
	if (!this.audio) return;
	this.lastSeekAt = Date.now();
	const clamped = Math.max(0, Math.min(1, frac));
	if (Number.isFinite(this.audio.duration) && this.audio.duration > 0) {
		this.audio.currentTime = clamped * this.audio.duration;
		this.pendingSeek = null;
	} else { this.pendingSeekFrac = clamped; }
	if (this.audio.paused) {
		void this.audio.play().catch(() => { /* autoplay rejected */ });  // ← D-03 free
	}
}
```
No store edit is expected this phase.

**Source:** `src/lib/actions/longpress.ts` (the inverse case — it SUPPRESSES the trailing click via a capture-phase guard; the lyric tap WANTS the click). Key idiom: a real drag moves > slop (8px) and the browser fires `pointercancel`/no `click`; a sub-slop tap fires `click`.

The `.lyrics` handler (`NowPlaying.svelte:122`) has NO `setPointerCapture`, so a genuine tap still produces a native `click` after pointerup — verified. **Apply to:** end spacer sizing (reuse the same `visHeight`) and any tap re-center (let the existing `$effect` re-run on the `autoScroll` flip — do NOT add a separate scrollIntoView).
