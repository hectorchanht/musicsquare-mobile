<objective>
Make playback survive screen-lock/backgrounding with a system media notification and lock-screen transport controls (D-04), by bridging the existing media-session seam to a native Capacitor plugin (D-05) without ever reloading the WebView on resume (D-06). This is the phase's defining technical risk: the Android System WebView has NO `navigator.mediaSession`, so the existing web wiring is a silent no-op on Android, and the OS kills backgrounded WebView audio within minutes without a `mediaPlayback` foreground service. The leading Web-API-shaped plugin (`@jofr/capacitor-media-session` v4) is stale on Cap 8, so the plan opens with a throwaway compat spike before any bridge code is written.

Purpose: An APK without background audio fails the project core value ("keep playing when the screen locks"). D-04 is non-negotiable.

Output: a confirmed plugin choice, a platform-branched `ms` accessor bridging to the plugin, FGS manifest/gradle wiring, a resume-safe lifecycle, and a device-test-matrix sign-off.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| native plugin ↔ WebView JS | The plugin renders the OS media UI and forwards transport actions back into JS player.* (cross-bridge control) |
| deep link / appUrlOpen → WebView routing | An external URL could attempt to re-navigate the WebView (D-06 / Tampering surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-999.1-10 | Denial of Service | foreground service / WebView killed while backgrounded | mitigate | FGS type `mediaPlayback` + FGS permissions keep the process alive; device matrix row 1 (5+ min lock) is the explicit gate |
| T-999.1-11 | Tampering | malicious deep link re-navigates the WebView (also a D-06 regression) | mitigate | Validate any `appUrlOpen` URL before routing; never `invalidateAll()`/navigate on resume; device matrix row 3 confirms no reload |
| T-999.1-12 | Tampering | adopting a stale/unverified background-audio plugin | mitigate | Task 1 [ASSUMED]-package gate (blocking-human, never auto-approved): verify both candidates on npmjs.com + a Cap-8 build spike before adopt; @mediagrid native-audio removed (conflicts with D-04/D-05) |
| T-999.1-13 | Information Disclosure | cleartext media/api traffic | mitigate | androidScheme https + `allowMixedContent: false` (Plan 02 capacitor.config.ts); all /api + CDN over https |
