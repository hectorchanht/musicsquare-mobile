## Task 1: Add Android APK build & install section to README.md

**Files:** README.md

**Action:**
Read README.md first; match its existing tone/structure and insert the new section in a
sensible spot (after dev/build docs if present, else near the end). Content to cover —
all facts verified during phase 999.1 execution, do not invent alternatives:

1. **Prerequisites**
   - Node v22 via nvm (`export PATH="$HOME/.nvm/versions/node/v22.22.0/bin:$PATH"` or `nvm use 22`), pnpm
   - JDK 21 — `brew install openjdk@21`, `export JAVA_HOME=/opt/homebrew/opt/openjdk@21` (JDK ≤20 fails with `invalid source release: 21`)
   - Android SDK — `export ANDROID_HOME=$HOME/Library/Android/sdk` (platform 36 auto-downloads on first build)
   - Note: `android/local.properties` is generated locally (sdk.dir) and must never be committed (gitignored)

2. **Build the APK**
   ```bash
   pnpm install
   pnpm build:native          # static SPA build, bakes VITE_API_BASE=https://openmusic.lol
   npx cap sync android
   cd android && ./gradlew assembleDebug
   ```
   Output: `android/app/build/outputs/apk/debug/app-debug.apk`

3. **Install to phone**
   - Enable Developer options + USB debugging on the phone, plug in, accept the RSA "Allow USB debugging?" prompt
   - `adb install -r --user 0 android/app/build/outputs/apk/debug/app-debug.apk`
     (`--user 0` required on Samsung devices with Secure Folder / DUAL_APP profiles — plain install can land in the wrong user)
   - Launch: `adb shell am start -n com.openmusic.app/.MainActivity`

4. **Important: server must be current**
   The APK loads its UI locally but calls `https://openmusic.lol/api/*` cross-origin.
   The deployed web app must include the current CORS hook, or every API call fails with
   a CORS error. Deploy is manual:
   ```bash
   pnpm build && npx wrangler pages deploy .svelte-kit/cloudflare --project-name openmusic
   ```

**Verify:** README renders correctly (markdown lint by eye); commands match package.json scripts (`build:native` exists); no `local.properties` content leaked.

**Done:** README.md contains the new section with all four sub-parts above.
