## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| resolved/cached cover URL → rendered surface | a cover URL string crosses into the DOM render path; it must be https-only and never widen the injection surface |
| Deezer/iTunes/CN proxy → cover resolve | untrusted upstream image URLs cross into cache + render |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-03 (T-0bb-01) | Tampering / XSS | cover-cache write + lazyCover resolve | mitigate | Only SOLID https URLs written (`isSolidCover` gate + `writeKey` empty no-op); resolved covers are URL strings consumed by callers as `<img src>` attributes, NEVER routed into CSS `url()` for resolved values |
| T-21-04 | Tampering | cover-cache localStorage poisoning | mitigate | writeKey no-ops on empty; values are URL strings only; quota errors degrade to no-op; clear-cover-cache recovery path exists (D-16) |
| T-21-05 | DoS | lazyCover unbounded fan-out | mitigate | unobserve-after-first + one-shot done flag + in-flight Set de-dupe + cache-first read before any network; per-row resolve is viewport-bounded; never Promise.all (reuse mapWithConcurrency CAP if batched) |
| T-21-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs (zero new deps — RESEARCH.md Package Legitimacy Audit: "Not applicable"); IntersectionObserver + Image are browser-native; no install task to gate |
