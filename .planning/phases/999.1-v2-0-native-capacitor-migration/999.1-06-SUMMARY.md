# Phase 999.1 Plan 06: Public Music/ MediaStore Bridge Summary

Delivered the public-`Music/` half of D-11 that plan 03 deferred: a **hand-written local Kotlin
Capacitor plugin** (`MediaStoreSaverPlugin.kt`, zero npm/git dependency ‚Äî the T-999.1-07
supply-chain mitigation that kept the rejected community `capacitor-mediastore` plugin out) that
inserts downloaded audio into the public `Music/OpenMusic/` collection via `MediaStore.Audio.Media`
(API 29+ `RELATIVE_PATH` + `IS_PENDING`; API ‚â§28 legacy `DIRECTORY_MUSIC` + media-scan). `blob-store`
native `put`/`del` now route through it: `put` keeps the plan-03 app-private copy (the offline-read
source for `get`) AND writes a public copy, recording the bridge's content URI; `del` removes both
and clears the index. The plan-03 `// TODO(999.1-06)` marker is removed. `assembleDebug` builds the
APK clean; the full 764-test suite + `pnpm check` stay green; the blobStore namespace + signatures
are byte-identical so all 4 consumers are untouched. **Task 3 (the device test matrix) is a BLOCKING
human-verify gate and has NOT been signed off ‚Äî no physical device is attached via adb, and
public-`Music/` visibility / offline playback / backgrounding survival require a human-held device
(RESEARCH: no CI/emulator fallback).**

## What Was Built

**Task 1 ‚Äî Hand-written Kotlin MediaStore plugin + registration + manifest (D-11) [commit e3ecb1b]**

- `android/app/src/main/java/com/openmusic/app/MediaStoreSaverPlugin.kt` ‚Äî `@CapacitorPlugin(name =
  "MediaStoreSaver")` extending `com.getcapacitor.Plugin`, package `com.openmusic.app`. Two
  `@PluginMethod`s:
  - `saveToMusic(call)`: reads `fileName` (required) + `base64` (required, `Base64.decode`d to bytes),
    infers the MIME from the extension (mp3‚Üíaudio/mpeg, flac‚Üíaudio/flac, m4a/aac‚Üíaudio/mp4,
    ogg‚Üíaudio/ogg, default audio/mpeg). **API 29+:** `ContentValues` with `DISPLAY_NAME`, `MIME_TYPE`,
    `RELATIVE_PATH = "Music/OpenMusic/"`, `IS_PENDING = 1`; `resolver.insert(getContentUri(
    VOLUME_EXTERNAL_PRIMARY), values)` ‚Üí `openOutputStream(uri).use { write(bytes) }` ‚Üí clear
    `IS_PENDING` via `update` ‚Üí `resolve({ uri })`. **API ‚â§28:** writes to
    `getExternalStoragePublicDirectory(DIRECTORY_MUSIC)/OpenMusic/<fileName>` (mkdirs) +
    `MediaScannerConnection.scanFile` so it is indexed ‚Üí resolve the `file://` URI.
  - `deleteFromMusic(call)`: `uri` arg, branches on scheme ‚Äî `content://` ‚Üí `contentResolver.delete`;
    `file://` ‚Üí delete the file + re-scan. Missing uri / not-found swallowed into `resolve()`.
  - Both bodies are try/catch ‚Üí `call.reject(message)` (TS maps reject to the never-throws sentinel).
- `MainActivity.java`: `registerPlugin(MediaStoreSaverPlugin.class)` inside `onCreate` BEFORE
  `super.onCreate` (Capacitor 8 registration).
- `AndroidManifest.xml`: added `WRITE_EXTERNAL_STORAGE android:maxSdkVersion="28"` (legacy branch
  only; no-op on API 29+ ‚Äî T-999.1-18). Plan-04 FGS perms/service untouched. NO READ_MEDIA_AUDIO /
  broad scope.
- **NO new npm/git dependency** (`git diff package.json` empty) ‚Äî hand-written Kotlin only.
- `./gradlew assembleDebug` ‚Üí BUILD SUCCESSFUL; `app-debug.apk` (5.2 MB) produced under Cap 8.4.0 /
  AGP 8.13 / Kotlin 2.2.20 / JDK 21.

**Task 2 ‚Äî blob-store native put/del through the bridge + TS wrapper (D-11) [TDD; RED 31adf8e, GREEN 98021b9]**

- `src/lib/services/media-store.ts` (new): `registerPlugin<MediaStoreSaverPlugin>('MediaStoreSaver')`
  with a typed interface for `saveToMusic({ fileName, base64 }): Promise<{ uri }>` and
  `deleteFromMusic({ uri }): Promise<void>`.
- `src/lib/services/blob-store.ts`: imported `MediaStoreSaver`; `nativePut` now (1) writes the
  app-private offline copy via `write_blob` (unchanged ‚Äî the `get()` read source) and (2)
  base64-encodes the Blob and calls `MediaStoreSaver.saveToMusic`, recording the returned content URI
  in `localStorage` under `openmusic-blob-uri:<uid>`. `nativeDel` deletes the app-private copy AND, if
  a URI was recorded, calls `MediaStoreSaver.deleteFromMusic({ uri })` then clears the index. `nativeGet`
  is unchanged (reads the app-private copy). Every bridge call is wrapped so `put‚Üífalse` / `del‚Üívoid`
  on any reject/throw ‚Äî never-throws parity preserved. The plan-03 `// TODO(999.1-06)` marker is gone.
  `export const blobStore = { put, get, del }` + the three signatures are byte-identical.
- `src/lib/services/blob-store.test.ts`: mocked `./media-store` + a `localStorage` shim; +4 cases ‚Äî
  put routes through `saveToMusic` with a base64 payload and records the URI; put resolves false when
  `saveToMusic` rejects; del calls `deleteFromMusic` with the recorded URI + clears the index; del
  resolves void when `deleteFromMusic` throws. Plan-03 cases kept green.

## Deviations from Plan

### [Rule 3 ‚Äî Blocking issue] Kotlin Gradle support absent in the cap-add Android project
- **Found during:** Task 1 `./gradlew assembleDebug` ‚Äî `compileDebugJavaWithJavac` failed:
  `cannot find symbol class MediaStoreSaverPlugin`. The cap-add-generated app module is pure Java with
  NO Kotlin Gradle plugin, so the mandated `.kt` plugin could not compile.
- **Fix:** Added `org.jetbrains.kotlin:kotlin-gradle-plugin:2.2.20` to the ROOT `android/build.gradle`
  buildscript classpath and applied `org.jetbrains.kotlin.android` to `:app` via a `project(':app') {
  apply plugin: ... }` block in the root file ‚Äî so `android/app/build.gradle` (owned by the concurrent
  Plan 05 executor) was NOT touched. (`afterEvaluate` failed with `KotlinPluginLifecycle cannot be
  started in ProjectState EXECUTED`; applying during `:app` configuration works.)
- **Files:** android/build.gradle ‚Äî **Commit:** e3ecb1b
- **Coordination note:** Plan 05 owns `android/app/build.gradle`; this fix lives entirely in the root
  `android/build.gradle` (not in either plan's `files_modified`) to avoid a merge collision. If Plan 05
  independently adds Kotlin support to the app module, the two are compatible (the `if
  (!plugins.hasPlugin(...))`-equivalent is not needed because Gradle de-dupes a plugin applied once).

No bugs (Rule 1). No missing critical functionality beyond Rule 3. No architectural (Rule 4) changes.
No auth gates. Zero new npm/git dependency, so no package-legitimacy checkpoint was required.
