<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `+server.ts` → JOOX upstream | Server injects the JOOX secret from `platform.env` into the upstream URL — the secret crosses ONLY server→upstream, never server→client |
| build pipeline → client bundle | The JOOX token must be statically absent from `.svelte-kit/output/client` |
| browser → `/api/joox/*` | Untrusted query; no token is ever supplied by the client |
| JOOX upstream detail (positional `n`) → identity | Position-keyed detail can return the wrong song after reorder/paginate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Information Disclosure | JOOX token in client bundle / network | mitigate | Token read only from `platform.env` server-side, injected into the upstream URL; build-grep proves absence from client bundle; `.dev.vars` gitignored (Task 2) |
| T-01-07 | Information Disclosure | JOOX token / signed audio params in Worker logs / `wrangler tail` | mitigate | No `console.log` of token or signed params in `proxy/joox.ts` or the route; Security V7 redaction enforced (Task 2) |
| T-01-08 | Tampering (identity) | JOOX positional `n` returning the wrong song | mitigate | Stable `joox:${songmid}` uid + re-validate returned songmid/title; throw loudly on mismatch (Task 1 Tests 2-3, criterion #4) |
| T-01-05c | Tampering | JOOX upstream body (Chinese-field contract drift) | mitigate | Adapter validates expected fields/songmid; treats mismatch as a typed error, not trusted data |
| T-01-03 | Tampering (XSS) | JOOX titles/lyrics (Chinese fields) | mitigate | Returned as plain strings into Track; never `{@html}`; Svelte auto-escapes downstream |
</threat_model>
