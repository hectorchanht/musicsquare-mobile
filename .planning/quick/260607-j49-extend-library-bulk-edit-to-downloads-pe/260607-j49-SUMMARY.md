# Quick Task 260607-j49 — Bulk-edit extended to Downloads + Playlists

Follow-on to ii6 which shipped bulk-edit for the Liked tab v1 with `rowAction()` architected
for this exact extension.

## Changes (single file: `src/routes/(app)/library/+page.svelte`)
- `rowAction(track, list, playlistId?)` now dispatches by current tab:
  - `liked` → `library.toggleLike(track)` (removes when already liked)
  - `downloads` → `library.removeDownload(track.uid)`
  - `playlists` → `library.removeFromPlaylist(playlistId, track.uid)`
- New `editableTabHasContent` derived shows the Edit pill on any of the 3 bulk-editable
  tabs when their underlying list has content. History is intentionally **left out** —
  its own "Clear all" button is the destructive path; per-row remove of stale history
  isn't useful.
- Downloads + each playlist row now wear `class:edit-row` + render `Trash2` in edit mode
  (mirrors the Liked pattern).
- Tab switch still auto-exits edit mode (`$effect` on `tab`).
- Downloads' inline per-row `.del` button (one-off precise remove) is hidden DURING edit
  mode so the bulk-edit affordance is unambiguous. Outside edit mode it's still there.
- Playlist-head `.del` (delete whole playlist) is left alone — different destructive
  action shouldn't ride the bulk-edit toggle.

## Verification (live preview, seeded library: 3 downloads, 1 playlist × 2 tracks)
- **Downloads**: Edit shows. Click Edit → 3 rows go `.edit-row`, inline `.del` buttons
  disappear (`inlineDelHidden: 0`). Tap a row → **3→2 rows**.
- **Playlists**: Edit shows when any playlist has tracks. Click Edit → 2 rows
  go `.edit-row`. Tap a row → **2→1 tracks**.
