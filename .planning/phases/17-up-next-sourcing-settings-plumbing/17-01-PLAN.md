<objective>
Introduce per-context up-next sourcing on top of the existing never-stop generation engine, and fix the track-change auto-expand bug. This is the Phase 17 foundation: it adds the `queueContext` `$state` field (which Phase 16 never shipped), threads a context argument through every play-entry call site, resolves a per-context sourcing mode (`'same-list' | 'generated'`, global default `'generated'`) from `UPNEXT_DEFAULTS`, and guards `autoExpandOnPlay` behind the `fresh` flag.

Purpose: Playing a song must fill up-next predictably — genre-generated by default, search results NOT silently appended — with the nowbar staying put on track change. This unblocks Plans 02/03/04 (which share `player.svelte.ts` / `settings.svelte.ts`).
Output: New `UPNEXT_DEFAULTS` config group, `queueContext`/context-arg plumbing on the player, `effectiveUpnextMode` resolver + persisted per-context map in settings, Settings → Playback selector rows, context args at the home/library/search call sites, extended unit tests, and ALL of Phase 17's new i18n keys batched across all 15 locales.

**i18n ownership note:** This plan is the SOLE owner of the i18n locale files for Phase 17. Task 4 adds EVERY new key the phase needs (up-next sourcing, `nowplaying.clearQueue`, `settings.demoPrefix`, the `deezer.*` keys) in one batch. Plans 02/03/04 only CONSUME those keys via `t(...)` and do NOT edit any locale file — this is what lets Plans 02/03/04 run in parallel (Wave 2) without a same-wave i18n write conflict.
</objective>
