# Quick Task 260606-3f6 — Summary

Six UX/behavior upgrades to the deployed MusicSquare app. Live at openmusic.pages.dev. Phase-1 data layer + its 58 tests untouched.

## Shipped

1. **Hide source.** Removed NetEase/QQ/Kuwo/JOOX badges/labels from home tiles, search rows, artist rows, and the NowPlaying meta + queue/related rows (and their now-unused CSS/consts). `track.source` stays internal. `/spike` dev tool keeps its source column.
2. **Cross-source dedupe + best quality.** New `src/lib/services/dedupe.ts` → `dedupeBest(tracks)`: groups by normalized title+artist (case/punct/space-insensitive, drops "(Live)"/"- Remaster" suffixes, CJK-safe), keeps the highest-quality variant (quality-string rank → source-preference tie-break), preserves first-seen order. Applied to home picks, search, artist hit-songs, NowPlaying Related, and the player queue.
3. **Shared picks builder.** New `src/lib/services/picks.ts` → `buildDiversePicks(count, excludeUids)` (artist pool + allSettled-per-artist + top-each + dedupe). Home uses it (cache + Randomize preserved); the player queue reuses it for auto-grow.
4. **Draggable sub-nav sheet (NowPlaying).** The Up-Next/Lyrics/Related sheet has a grab handle: drag **up** → fills the screen (covers cover/controls); drag **down** → restores the now-playing view (double-tap toggles too). Separate from the existing cover-drag-down-to-collapse gesture.
5. **Smart lyrics scroll.** Active line auto-scrolls to center as the song plays; touching/wheeling the lyrics pauses auto-scroll so you can browse; ~2.5s after you stop, auto-scroll resumes and re-centers the active line. (`scrollIntoView` gated by an `autoScroll` flag + idle timer.)
6. **Auto-advance + auto-growing queue.** `ended` → `next()` (audio lives in root layout so it survives nav). Player store `ensureAhead()` appends fresh `buildDiversePicks(...)` (excludes existing uids, dedupes) whenever the current track is within 2 of the queue end — guarded by a `growing` flag + tolerates dry sources. So the list never runs out.
