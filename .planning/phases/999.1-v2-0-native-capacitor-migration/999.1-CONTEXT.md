# Phase 999.1: v2.0 Native (Capacitor) migration — Context

**Gathered:** 2026-06-11
**Status:** Ready for planning
**Scope refinement:** User narrowed this backlog item to an **Android-only quick-APK slice** — "android app quickly now, just apk, no play store". iOS portion deferred.

<domain>
## Phase Boundary

Produce an installable, signed Android APK of MusicSquare via Capacitor, distributed by sideload (GitHub Releases / Obtainium) — no Play Store, no store-compliance work. Single frontend codebase: web deployment stays untouched. iOS, and API extraction to a standalone worker, are explicitly out of this slice.

</domain>

<decisions>
## Implementation Decisions

### Wrapper strategy
- **D-01:** Bundled static build — `BUILD_TARGET=native` env switches `svelte.config.js` to `adapter-static` with SPA fallback; default stays `adapter-cloudflare` so web deploys are unchanged.
- **D-02:** No API worker extraction this phase. Native build gets an `API_BASE` config pointing at the **existing deployed Pages app's `/api/*` routes**; add CORS handling to those routes (allowlist Capacitor origins: `https://localhost`, `http://localhost`, `capacitor://localhost`).
- **D-03:** `API_BASE` empty = same-origin (web build unchanged); centralize in one fetch helper rather than touching every source adapter (per assessment doc).

### Background audio (must-work, full)
- **D-04:** Background audio is in scope and non-negotiable: playback must survive screen lock/backgrounding via Android foreground service + media notification plugin, with lock-screen controls (play/pause/next/prev). An APK without this fails the core value.
- **D-05:** Bridge to the existing MediaSession service in the web code where possible; plugin selection is research-phase work.
- **D-06:** Never reload/refresh the WebView on app resume — same resume-intent rule already enforced on web (see memory: manifest `launch_handler: focus-existing` fix). Capacitor activity must restore, not re-navigate.

### Build, signing & updates
- **D-07:** GitHub Actions release pipeline from day one: tag push → workflow builds, signs (release keystore in repo secrets), publishes APK to GitHub Releases.
- **D-08:** Update channel = Obtainium pointed at GitHub Releases. Requires consistent versionName/versionCode bumping per release (scheme at Claude's discretion, derived from git tag).
- **D-09:** Releases publish on the `origin` GitHub repo (pushes as `hectorchanht` via SSH host `github-b`).

### Downloads / offline storage
- **D-10:** Filesystem swap happens now (not deferred): platform-switch behind the existing `src/lib/services/blob-store.ts` interface — IndexedDB on web (unchanged), `@capacitor/filesystem` + `capacitor-blob-writer` on native.
- **D-11:** Downloaded songs go to the **public Music folder** (user-visible, readable by other apps). Accept the extra MediaStore/scoped-storage work and Android-version quirks this entails; metadata index stays in localStorage.

### Claude's Discretion
- SPA fallback/prerender details for the static build
- Exact CORS implementation in the SvelteKit app (hooks vs per-route)
- Media-controls/foreground-service plugin choice (research decides)
- versionName/versionCode scheme, min SDK level
- App icon/splash, status-bar/safe-area handling

</decisions>

<specifics>
## Specific Ideas

- "Quickly — just APK, no Play Store": bias every choice toward shortest path to an installable APK; the only deliberate scope-adds are full background audio and the filesystem download swap, which the user explicitly opted into.
- Player must behave like the web app on resume: no refresh on re-entry, ever (except OS kill).

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `src/lib/services/blob-store.ts` — clean blob interface seam; native filesystem backend slots behind it.
- Existing MediaSession service — already drives lock-screen metadata on web; bridge target for native media notification.
- `src/routes/api/*` (source proxy, deezer, fivesing, jamendo, lastfm, similar, translate) — stay where they are; native build calls them cross-origin via `API_BASE`.

### Established Patterns
- Svelte 5 runes, store classes for settings/library in localStorage — all of this works unchanged inside the Capacitor WebView app container (no Safari eviction problem).
- Tooling gotcha: shell default node is v16 — prefix PATH with nvm v22.22.0 and use pnpm (including in subagent prompts).

### Integration Points
- `svelte.config.js` — `BUILD_TARGET` adapter switch.
- One fetch helper for `API_BASE` (to be introduced) — single point where all `/api/...` calls route.
- SvelteKit server hooks/routes — CORS allowlist for Capacitor origins.
- New: `/android` platform dir + `capacitor.config.ts`; GitHub Actions workflow for signed-release APK.

</code_context>

<deferred>
## Deferred Ideas
