# Quick Task 260605-wq1: Sketch-001 variant C home shell → real app + deploy

## Description
Implement the sketch-001 "MusicSquare-branded" (variant C) home shell as the real app, wired to the live Phase 1 data layer (`catalog.searchAll` / `ensureTrackDetails`), then build and deploy to the `openmusic` Cloudflare Pages project. Off the planned phase order (this is Phase-4-shaped UI pulled forward as a demo); a basic `<audio>` covers playback (full engine is Phase 6).

## Tasks
1. **Theme + player store** — `src/app.css` (variant C tokens: dark, violet #7c5cff) + `src/lib/stores/player.svelte.ts` (Svelte 5 runes singleton: current track, play/pause, single `<audio>` attach, `ensureTrackDetails` on play, browser-direct audio, `referrerpolicy=no-referrer`).
2. **App shell** — `src/routes/(app)/+layout.svelte`: persistent glassy now-playing bar + bottom tab nav (Home/Search/Library) + the single `<audio>`; renders page children. Scoped to a route group so `/spike` stays bare. Root `+layout.svelte` imports `app.css`.
3. **Home** — `src/routes/(app)/+page.svelte`: top nav (brand + full search pill → /search), 3×3 grid of 9 songs from `searchAll` (random seed keyword), source badge + quality + cover (gradient fallback), tap → `player.play`.
4. **Search** — `src/routes/(app)/search/+page.svelte`: input → `searchAll` → interleaved result list → tap to play. Remove placeholder root `+page.svelte` (conflicts with `(app)/+page.svelte` for `/`).
5. **Verify + deploy** — `pnpm check` strict clean, `pnpm build` ok, then `wrangler pages deploy .svelte-kit/cloudflare --project-name openmusic`.

## Constraints
- Do NOT touch the data layer / proxy / adapters (reuse Track, registry, catalog, lrc).
- No new heavy deps — Svelte 5 runes + built-in transitions only.
- Mobile-first responsive.

## must_haves
- `/` renders the variant-C home shell with a 3×3 grid populated from the real 4-source data layer.
- Tapping a tile resolves details and plays browser-direct; now-playing bar reflects current track with working play/pause.
- Search route returns interleaved cross-source results and plays them.
- `pnpm check` + `pnpm build` pass; deployed to openmusic.pages.dev.
