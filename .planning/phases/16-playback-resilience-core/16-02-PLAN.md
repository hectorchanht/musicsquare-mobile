<objective>
Add the never-stop resilience policy on top of the existing failover/prefetch engine in `player.svelte.ts` (+ a thin offline gate in `fallback.ts`). This is the dependency root of v1.2: a consecutive-failure counter, a ~5-skip loop-guard stop state, an offline short-circuit that switches up-next to downloads, a ~15s initial-load stall watchdog that routes into failover, rejected-`play()`-as-failure accounting, prefetch on the ended→next path, and a store-level `notice` channel (sticky message + optional Retry action) for the layout toast host (16-03) to consume.

Purpose: PLAY-07 (all-source retry → skip), PLAY-08 (never-stop except sleep/offline/loop-guard; rejected play() counts as failure), PLAY-09 (prefetch-on-track-change). Implements CONTEXT decisions D-02 (skip + batched-skip notice), D-04 (loop-guard stop + sticky Retry), D-05 (recovery = skip-ahead + reset + re-arm), D-06 (playing-event reset; rejected play() = failure), D-07 (offline auto-switch to downloads), D-08 (offline gate; no counter burn; offline pause notice), D-09 (this phase owns the player offline switch; Phase 24 owns the shell), D-12 (repeat-one breaks on failing loop), D-13 (~15s initial-load stall → failover), D-14 (mid-track stall = buffering only, no failover), D-15 (prefetch = URL resolve-ahead; single-element invariant).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → music-source proxies (/api/*) | Failover/skip/stall paths fan out repeated `searchAll`/`ensureTrackDetails` calls to no-SLA third-party proxies; an uncapped loop is a self-inflicted DoS / IP-ban risk. |
| navigator.onLine → failure chain | Browser online state gates whether the network failover runs at all. |
| IndexedDB blob store → audio.src | Downloaded blobs (object URLs) feed the offline up-next switch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03 | Denial of Service | runFallback / skip chain (self-DoS against no-SLA proxies) | mitigate | The consecutive-failure counter caps auto-skips at 5 then STOPS (no further network resolve until a user gesture); the existing AbortController + 200ms watchdog cancels stale in-flight searches; the offline gate skips the network entirely when offline. This is the primary mitigation for Pitfall 1 (toast storm / proxy 429 / battery drain). |
| T-16-04 | Tampering | manual-skip race clobbering user choice | mitigate | Every advance routes through next()→play() which bumps playGen; every async continuation re-checks `playGen` after each await and discards stale results (Pitfall 2). No parallel fast-skip path is added. |
